# 컨텍스트 압축 (2026-04 업데이트)

## 목표
대화가 길어져도 핵심만 추려서 토큰 최소화
데이터는 절대 삭제하지 않는다 — 압축만 한다

---

## 언제 압축하나

| 상황 | 조치 |
|------|------|
| 컨텍스트 50% 도달 | /compact 즉시 실행 |
| 큰 마일스톤 완료 직후 | /compact로 깔끔한 재시작 |
| memory/context.md 10개 초과 | 오래된 것부터 한 줄 요약 |
| 새 세션 시작 시 | 이전 세션 컨텍스트 누적 없이 시작 |

## /compact 실행 전략
- 꽉 차기 전에 선제적으로 (50%에서)
- 자동 압축보다 수동 /compact가 더 정확한 요약 생성
- 복잡한 작업은 세션 앞 80%에서 처리, 뒤 20%는 가벼운 작업만

## memory/ 파일 압축 규칙

### context.md
- 10개 초과 시 → 오래된 것부터 한 줄 요약
- 형식: `[날짜] [주제] [점수] [한 줄 메모]`
- 예: `260328 AI자동화 9점 훅반전형성공`
- 원본은 Supabase에 있으니 로컬 삭제 안전

### 캘린더.md
- 지난 날짜 항목 → 삭제 (Supabase 백업됨)
- 이번 주 + 다음 주만 유지

## brain/ 파일 조건부 로드 (토큰 절약 핵심)
항상 로드 (필수 3개만):
- brain/인증.md
- brain/파일맵.md
- brain/Claude코드제한.md

조건부 로드 (필요할 때만):
- 콘텐츠 생성 시 → 판단 + 개인화 + 바이럴공식
- 월요일만 → AI모니터 + 트렌드예보 + 정책모니터
- 5회마다 → 자기학습
- 10회마다 → 댓글분석 + 예측엔진 + 피드백루프
- 에러 시 → 에러복구
- 보안 이슈 → 보안 + 법적준수

**절대 금지**: 모든 brain/ 파일 한꺼번에 읽기

## 실행 방법
1. `/compact` 입력 → Claude가 대화 요약 생성
2. 요약 내용 확인 후 계속 진행
3. 핵심 컨텍스트 손실 없이 토큰 50% 절감
