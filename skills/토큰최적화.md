# 토큰 최적화 고도화 (2026-04 업데이트)

## 핵심 원칙
데이터는 절대 삭제하지 않는다 — 압축만 한다

---

## 1. CLAUDE.md 최소화
- **2,000 토큰(약 1,500자) 이내** 유지 (매 요청마다 주입됨)
- 세부 규칙은 brain/ 파일에 위임, CLAUDE.md는 "링크"만
- 새 규칙 추가 시 기존 규칙과 합치기, 중복 표현 제거

## 2. 세션 격리 (Session Isolation)
- Claude Code를 하나의 긴 대화로 쓰지 말 것
- 작업 단위별로 새 세션 시작 → 이전 작업 컨텍스트 누적 방지
- 복잡한 작업은 세션 앞 80%에서 처리, 뒤 20%는 가벼운 작업만

## 3. /compact 전략적 실행
- 맥락이 50% 찼을 때 /compact 실행 (꽉 차기 전에 선제적으로)
- 큰 마일스톤 완료 직후 실행 → 깔끔한 요약으로 재시작
- 자동 압축보다 수동 /compact가 더 정확한 요약 생성

## 4. brain/ 파일 조건부 로드
항상 로드 (필수 3개만):
- brain/인증.md
- brain/파일맵.md
- brain/Claude코드제한.md

조건부 로드 (필요할 때만):
- 콘텐츠 생성 시 → 판단 + 개인화 + 바이럴공식
- 월요일만 → AI모니터 + 트렌드예보 + 정책모니터
- 5회마다 → 자기학습
- 10회마다 → 댓글분석 + 예측엔진 + 피드백루프
- 에러 시 → 에러복구
- 보안 이슈 → 보안 + 법적준수

**절대 동시 로드 금지**: 모든 brain/ 파일 한꺼번에 읽기 금지

## 5. memory/ 파일 압축 규칙
context.md:
- 10개 초과 시 → 오래된 것부터 한 줄 요약
- 형식: [날짜] [주제] [점수] [한 줄 메모]
- 예: 260328 AI자동화 9점 훅반전형성공
- 원본은 Supabase에 있으니 로컬 삭제 안전

캘린더.md:
- 지난 날짜 항목 → 자동 삭제 (Supabase 백업됨)
- 이번 주 + 다음 주만 유지

## 6. 명령어 범위 최소화
- "전체 테스트 실행" 대신 "auth 모듈 테스트만 실행"
- 셸 명령 출력 전체가 컨텍스트로 들어옴 → 범위 좁힐수록 토큰 절약
- MCP 툴이 대용량 JSON 반환 시 → context-mode로 인덱싱 (요약만 수신)

## 7. 프롬프트 캐싱 활용
- system prompt, CLAUDE.md 등 반복 내용 → Anthropic이 자동 캐싱
- 캐시 히트 시 토큰 비용 90% 절감
- 반복되는 규칙/컨텍스트는 앞쪽에 배치 (캐시 효율 최대화)

## 8. 모델 선택 전략
- Haiku: 단순 조회, 포맷팅, 빠른 확인
- Sonnet: 테스트 작성, 일반 코드 수정 (기본값)
- Opus: 복잡한 아키텍처 설계, 멀티파일 리팩토링

## 9. output/ 파일 정리
- 30일 이상 된 output/ → output/archive/ 이동
- 실시간 로드 대상에서 제외
- 필요할 때만 archive/ 에서 불러오기

## 10. 1M 컨텍스트 활용 (2026-03 기준)
- Sonnet/Opus 4.6: 1M 토큰 컨텍스트 GA (추가 비용 없음)
- 대용량 코드베이스 한 번에 넣기 가능
- 단, 긴 컨텍스트 = 느린 응답 → 필요할 때만 사용
