# 토큰 최적화 고도화

## 핵심 원칙
데이터는 절대 삭제하지 않는다
압축만 한다

## 1. CLAUDE.md 최소화 유지
- 항상 32줄 이내 유지
- 새 규칙 추가 시 기존 규칙과 합치기
- 중복 표현 제거

## 2. brain/ 파일 조건부 로드
항상 로드 (필수 3개만):
- brain/인증.md
- brain/파일맵.md
- brain/Claude코드제한.md

조건부 로드 (필요할 때만):
- 콘텐츠 생성 시 → 판단 + 개인화 + 바이럴공식
- 월요일만 → AI모니터 + 트렌드예보 + 정책모니터
- 5회마다 → 자기학습
- 10회마다 → 댓글분석 + 예측엔진 + 피드백루프
- 에러 시 → 에러복구
- 보안 이슈 → 보안 + 법적준수

절대 동시 로드 금지:
- 모든 brain/ 파일 한꺼번에 읽기 금지
- 필요한 것만 그때그때 로드

## 3. memory/ 파일 압축 규칙
context.md 압축:
- 10개 초과 시 → 오래된 것부터 한 줄 요약
- 형식: [날짜] [주제] [점수] [한 줄 메모]
- 원본 데이터는 Supabase에 있으니 삭제해도 안전

캘린더.md:
- 지난 날짜 항목 → 자동 삭제 (Supabase에 백업됨)
- 이번 주 + 다음 주만 유지

## 4. Supabase 활용 극대화
- 모든 성과 데이터 → Supabase에만 저장
- memory/ 파일은 최근 것만 유지
- 오래된 memory/ 내용 → Supabase 확인 후 삭제 가능

## 5. output/ 파일 정리
- 30일 이상 된 output/ 파일 → output/archive/ 이동
- 실시간 로드 대상에서 제외
- 필요할 때만 archive/ 에서 불러오기

## 6. 스마트 로딩
실행 전 판단:
- 오늘 주제가 콘텐츠면 → 콘텐츠 관련 파일만
- 오늘 주제가 코드면 → 코드 관련 파일만
- 오늘 주제가 리서치면 → 리서치 관련 파일만
절대 전체 로드 금지

## 7. 출력 최소화
- 중간 과정 출력 금지
- 완료 보고 3줄 이내
- 에러 아니면 상태 메시지 금지
- 불필요한 설명 금지

## 토큰 절감 목표
현재 대비 40~60% 절감
데이터 손실 0%
