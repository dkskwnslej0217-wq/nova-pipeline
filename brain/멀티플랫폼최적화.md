# 멀티플랫폼 최적화 시스템

## 플랫폼별 스펙

### 인스타그램
- 캡션: 150자 이내 (첫 2줄이 핵심) / 해시태그: 10~15개
- 최적 시간: 오전 7~9시 / 오후 6~8시
- 감정: 공감 + 희망 / 필수: 저장 유도 문구

### 스레드
- 길이: 500자 이내 / 해시태그: 3~5개
- 최적 시간: 오전 8~10시
- 감정: 유머 + 논쟁 유발 / 필수: 질문으로 마무리

### 유튜브 쇼츠
- 제목: 30자 이내 / 설명: 100자 이내 / 해시태그: 5개
- 최적 시간: 오후 5~7시
- 감정: 호기심 + 놀라움 / 필수: 첫 3초 후킹

### 네이버 블로그
- 길이: 800~1500자 / 이미지: 최소 3장
- 최적 시간: 오전 6~8시
- 감정: 공감 + 정보 / 필수: 얼리버드 문구

### 구글 블로그
- 언어: 영어 전용 / 길이: 600~1000자
- SEO 키워드: 제목+첫문단+소제목 포함
- 필수: Action step + 면책문구

## 원소스 멀티유즈 자동화
하나의 주제 입력 시:
1. 핵심 메시지 추출
2. 플랫폼별 스펙으로 자동 변환
3. 각 플랫폼 최적화 버전 동시 생성
4. 품질채점 후 발행 준비

## 발행 시간 스케줄러
memory/발행캘린더.md 자동 관리:

- 하루 최대 발행 수 초과 방지
- 동일 주제 중복 발행 방지
