# 멀티유저 시스템

## 목표
여러 사람이 각자 다른 설정으로 쓸 수 있는 구조

## 유저별 분리 항목
- 유저별 Supabase 테이블 (user_id 컬럼으로 구분)
- 유저별 MASTER_PASSWORD
- 유저별 성과 추적
- 유저별 개인화 패턴

## Supabase 테이블 업데이트
SQL Editor에서 실행:
alter table memory add column user_id text default 'default';

## 유저 등록 방법
.env 파일에 추가:
USER_ID=여기에_유저명_입력 (예: junho, user1)

## 유저별 실행
run_core.md 실행 시 USER_ID 기반으로
Supabase에서 해당 유저 데이터만 조회/저장

## 유저별 개인화
각 유저마다 별도 brain/개인화_[유저명].md 생성
성공 패턴이 유저별로 독립적으로 학습됨
