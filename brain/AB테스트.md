# A/B 실험 시스템

## 목표
훅/포맷/길이/주제방향 자동 실험해서
최고 성과 조합 찾기

## 실험 항목

### 훅 실험
A: 반전형 "X → Y"
B: 숫자형 "3가지", "5분 만에"
C: 질문형 "왜?", "어떻게?"
D: 공감형 "저도 몰랐어요"

### 포맷 실험
A: 짧고 강렬 (3문장 이내)
B: 스토리텔링 (기승전결)
C: 리스트형 (1,2,3 순서)
D: 문답형 (질문→답변)

### 길이 실험
A: 초단형 (30초 이내)
B: 단형 (1분)
C: 중형 (3분)

## 실험 방법
1. 매 콘텐츠 생성 시 각 항목별 랜덤 조합 선택
2. 동일 주제 A/B 2가지 버전 생성 → 품질채점 → 높은 버전 발행
3. 낮은 버전 memory/AB보관.md 저장
4. 조합 + score_total Supabase 기록
5. 20개 쌓이면 최고 성과 조합 자동 분석 → brain/개인화.md 반영

## 테스트 대상 추가
- 제목 패턴 / 썸네일 컨셉 / 발행 시간대 / 해시태그 조합 / 본문 구조

## 성과 추적 (발행 후 72시간)
클릭률 / 저장률 / 공유율 수집 → 승리 버전 패턴 brain/승리패턴.md 기록

## 승리 패턴 누적 (brain/승리패턴.md)
- 제목 이긴 패턴 TOP 10 / 반응 높은 시간대 TOP 3
- 효과적인 해시태그 조합 TOP 10 / 잘된 본문 구조 TOP 5
→ 이후 생성 시 자동 우선 적용

## 테스트 사이클
- 매 10회 생성마다 새 A/B 테스트 1개 자동 설계
- 월 4개 테스트 동시 진행
- 결과 memory/AB결과_[년월].md 저장
