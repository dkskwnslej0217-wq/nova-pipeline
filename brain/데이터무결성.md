# 데이터 무결성 보호

## Supabase 데이터 보호
- 저장 전: 민감정보 마스킹 (이름→익명, 전화→끝4자리만)
- 쿼리: 파라미터 바인딩 필수, 문자열 직접 삽입 금지
- 백업: 매일 1회 자동 (Supabase 기본 기능 활용)
- 삭제: 사용자 요청 시 72시간 내 완전 삭제

## 입력값 검증 규칙
모든 외부 입력에 적용:
- 길이 제한: 단일 입력 최대 2000자
- 특수문자 필터: <script> ; DROP TABLE -- 등 차단
- 인코딩: UTF-8만 허용
- 빈 입력: 처리 거부

## 출력 검증 규칙
응답 전송 전 체크:
- 개인정보 포함 여부
- 키/토큰 문자열 포함 여부
- 시스템 내부 경로 노출 여부
