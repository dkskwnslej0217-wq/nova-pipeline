# 에러 자동복구 시스템

## 에러 감지 즉시 실행 순서
1. 에러 내용 brain/보안로그.md에 기록
2. 아래 복구 플로우 자동 실행
3. 복구 성공 시 "자동복구 완료" 보고
4. 복구 실패 시 Telegram 알림 + 수동 개입 요청

## 에러별 자동복구 플로우

### API 에러 (Anthropic/Tavily/Groq)
- 1차: 3초 후 재시도
- 2차: 다른 모델로 자동 전환 (Anthropic→Groq→Gemini)
- 3차: 전환도 실패 시 "서비스 일시중단" 보고

### Supabase 연결 에러
- 1차: 5초 후 재시도 (최대 3회)
- 2차: memory/로컬캐시.md에서 임시 데이터 로드
- 3차: 3회 전부 실패 시 Telegram 알림

### 파일 없음 에러 (brain/skills/ 파일 못 찾을 때)
- 1차: memory/백업/ 에서 자동 복원
- 2차: 백업도 없으면 빈 파일 생성 후 보고
- 3차: CLAUDE.md 에 복원 실패 항목 기록

### Make.com 웹훅 에러
- 1차: 30초 후 재시도
- 2차: 실패 시 brain/보안로그.md 기록
- 3차: 3회 연속 실패 시 Telegram 알림

### 알 수 없는 에러
- 1차: 에러 메시지 전체 기록
- 2차: 마지막 성공 상태로 롤백 시도
- 3차: "수동 확인 필요" 보고

## 자동복구 성공 기준
- 동일 작업 재실행 시 에러 없음
- 데이터 손실 없음
- 3분 이내 복구 완료
