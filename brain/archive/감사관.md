# 감사관 시스템 (실수 감지 + 수정)

## 목표
시스템 전체를 감시하면서
실수, 충돌, 오류를 자동으로 감지하고 수정

## 감시 항목

### 1. 파일 충돌 감지
brain/ 파일들 서로 충돌하는 규칙 있는지 체크:
- 같은 기능 중복 정의 → 하나로 합치기
- 서로 반대되는 규칙 → junho한테 보고
- 삭제된 파일 참조 → 자동 수정

### 2. 규칙 일관성 체크
- CLAUDE.md 규칙 vs brain/ 파일 규칙 충돌 여부
- run_core.md 단계 순서 논리적인지 확인
- 조건부 로드 규칙 정확한지 확인

### 3. 데이터 무결성 체크
Supabase 데이터:
- 점수 범위 벗어난 값 (0~10 이외) → 자동 수정
- user_id 없는 기록 → default로 자동 설정
- 날짜 형식 오류 → 자동 수정

### 4. API 상태 체크
- Tavily API 응답 정상인지
- Supabase 연결 정상인지
- Groq API 응답 정상인지
- Make.com 웹훅 정상인지

### 5. 법적 준수 체크
매 콘텐츠 생성 후:
- 광고 표시 누락 여부
- 면책 문구 누락 여부
- 저작권 침해 요소 여부

### 6. 보안 체크
- .env 파일 정상인지
- API 키 유효한지
- 인증 시스템 정상인지

## 실수 발견 시 처리

### 자동 수정 가능한 것
- 데이터 형식 오류
- 중복 내용
- 누락된 면책 문구
→ 자동 수정 후 로그 기록

### junho 확인 필요한 것
- 규칙 충돌
- API 키 만료
- 법적 위반 가능성
→ 즉시 보고 + 해결방안 제시

### 즉시 중단해야 하는 것
- 보안 침해 시도
- 법적 위반 콘텐츠
- 시스템 전체 오류
→ 즉시 중단 + junho한테 긴급 보고
