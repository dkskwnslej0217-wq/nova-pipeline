# 사용자 패턴 학습 시스템

## 목표
사용자들이 실제로 쓰는 패턴 수집해서
시스템 자동 업그레이드

## 수집 항목
Supabase users 테이블에 아래 추가:
- 가장 많이 쓴 플랫폼
- 가장 많이 쓴 주제 유형
- 가장 많이 쓴 포맷
- 실패 많이 난 부분
- 자주 요청하는 기능

## 분석 방법
10명 이상 같은 패턴 발견 시:
- 많이 쓰는 기능 → brain/ 파일 자동 강화
- 안 쓰는 기능 → archive/ 이동 제안
- 새 요청 패턴 → 새 모듈 생성 제안

## 업그레이드 프로세스
1. Supabase에서 전체 유저 데이터 분석
2. 패턴 발견 → memory/사용자패턴.md 저장
3. 강화할 기능 목록 → junho한테 보고
4. junho 승인 → 자동 업그레이드 실행

## 실행 주기
매주 월요일 run_weekly.md에 포함
