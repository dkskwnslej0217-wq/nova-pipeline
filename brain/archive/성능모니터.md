# 성능 모니터링

## 목표
시스템 전체 성능 추적해서 병목 찾기

## 측정 항목 (Supabase performance 테이블 자동 기록)
- 실행 시간 (시작~완료) / 토큰 사용량 추정
- 사용 모델 (Anthropic/Groq/Gemini) / 성공·실패 여부
- 에러 발생 시 에러 종류
- 콘텐츠 생성 시 품질점수 (자기검증 점수)
- 평균 자기점수 추이 / 실패율 추이 / 가장 많이 로드되는 파일

## 성능 기준선 (목표)
- 실행시간: 30초 이내
- 성공률: 95% 이상
- 품질점수: 80점 이상
- 에러 복구율: 90% 이상

## 기준선 미달 시
- 실행시간 초과 → brain/토큰최적화.md 재적용
- 성공률 하락 → brain/에러자동복구.md 점검
- 품질점수 하락 → skills/ 파일 업그레이드 필요 보고
- 에러 복구율 하락 → brain/침해대응.md LEVEL 2 발동

## 분석 주기
- 매 10회 실행마다 자동 분석
- 매주 월요일: 평균 실행시간 / 성공률 / 최다 에러 / 품질점수 평균 / 전주 대비 3줄 보고
