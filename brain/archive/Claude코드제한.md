# Claude Code 제한 규칙

## 목표
Claude Code가 절대 하면 안 되는 것들을 명확히 정의

## 절대 금지 (즉시 중단)
- 시스템 파일 접근/수정 (C:\Windows 등)
- my-project 폴더 외부 파일 삭제
- 다른 사용자 데이터 접근
- 네트워크 포트 열기
- 외부 서버에 데이터 무단 전송
- .env 파일 내용 출력/공유
- API 키 콘솔에 출력
- 무한 루프 실행
- 과도한 API 호출 (1분에 10회 초과)

## 허용된 폴더만 접근
허용: my-project/ 내부 전체
금지: my-project/ 외부 모든 경로

## API 호출 제한
- Tavily: 분당 5회 이내
- Supabase: 분당 10회 이내
- Groq: 분당 10회 이내
- 초과 시 자동 대기 후 재시도

## 파일 작업 제한
- 한 번에 삭제 가능한 파일: 최대 3개
- 3개 초과 삭제 시 → 사람에게 확인 요청
- output/ 외부에 파일 생성 금지

## 자동 실행 제한
- 연속 실행: 최대 10회
- 10회 초과 시 → 사람에게 확인 요청
- 새벽 2시~6시 자동 실행 금지

## 위반 감지 시
1. 즉시 실행 중단
2. brain/보안로그.md에 기록
3. "제한 위반 감지: [내용]" 보고
4. 사람 확인 전까지 해당 작업 금지
