# 플랫폼 정책 모니터링

## 목표
유튜브/인스타/스레드 정책 변경 감지해서
시스템 자동 업데이트

## 모니터링 대상
Tavily로 매주 검색:
- "YouTube policy update 2026"
- "Instagram algorithm change 2026"
- "Threads API update 2026"
- "Claude API update 2026"
- "Supabase update 2026"
- "Tavily API update 2026"

## 감지 시 처리
### 콘텐츠 플랫폼 (유튜브/인스타/스레드)
- 업로드 한도 변경 → run_core.md 자동 수정
- 해시태그 정책 변경 → skills/출력포맷.md 수정
- 알고리즘 변경 → brain/바이럴공식.md 업데이트

### API 서비스 (Claude/Supabase/Tavily)
- 가격 변경 → brain/AI모니터.md 대안 탐색 실행
- 기능 변경 → 해당 skills/ 파일 자동 수정
- 서비스 종료 → 즉시 대안 탐색 + 교체 제안

## 변경 심각도 분류
- 낮음: 마이너 업데이트 → 메모만
- 중간: 기능 변경 → skills/ 파일 수정
- 높음: 정책 위반 가능성 → 즉시 실행 중단 + 보고
- 긴급: 서비스 종료 → 대안 즉시 탐색

## 자동 대응 한계
자동으로 못 하는 것:
- 새 API 가입 (사람이 직접)
- 법적 해석 (전문가 필요)
- 결제 변경 (사람이 직접)
→ 위 경우 "사람 확인 필요" 보고 후 대기
