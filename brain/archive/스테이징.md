# 스테이징 시스템

## 구조
생성 → output/staging/ 대기 → "업로드 시작해줘" → 재구성 → Make.com → output/uploaded/

## 파일 형식
파일명: `[날짜]_[플랫폼]_[주제].md`
```
---
platform: threads|instagram|youtube
topic: [주제]
score: [점수]
status: staging
created: [날짜]
---
[본문]
```

## 트리거별 동작

### "업로드 시작해줘"
1. staging/ 전체 읽기 → 플랫폼별 분류
2. score≥7 → 업로드대상 / score<7 → junho확인요청
3. 플랫폼 포맷 적용 (스레드/인스타/유튜브 각 _v4.md)
4. 업로드 제한: YT 하루5개·IG 하루3개 초과시 다음날 예약 / 새벽2~6시 금지
5. Make.com 웹훅 호출 ($MAKE_THREADS/INSTAGRAM/YOUTUBE_WEBHOOK)
6. 성공→status:uploaded→uploaded/ 이동 / 실패→에러알림.md→텔레그램
7. 보고: `완료 Threads N / Instagram N / YouTube N | 실패 N | 대기 N`

### "스테이징 목록"
`[플랫폼] [날짜] [주제] [점수] [상태]` 형식으로 출력

### "스테이징 비워줘"
staging/ 전체 → uploaded/ 이동 (업로드 없이 보관)

## 자동 스테이징 조건 (run_core.md 23단계)
score≥7 → staging/ 자동저장 / score<7 → output/만 저장
