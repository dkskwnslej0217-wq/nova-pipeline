# 사용자 피드백 자동 업데이트

## 목표
사용자가 불편한 사항 말하면
자동으로 해당 파일 업데이트

## 피드백 수집 방법

### 방법 1 — 직접 입력
"피드백: [내용]" 명령 시 즉시 처리

### 방법 2 — Make.com 연동
구글폼 or 카카오톡 → Make.com 웹훅
→ Claude Code가 자동으로 처리

### 방법 3 — 댓글 수집
brain/댓글분석.md와 연동
자주 나오는 불만 자동 감지

## 피드백 처리 프로세스
1. 피드백 내용 분석
2. 어떤 파일 수정이 필요한지 판단
3. 자동으로 해당 파일 수정
4. 수정 내용 memory/피드백업데이트로그.md에 기록
5. "업데이트 완료: [내용]" 보고

## 피드백 유형별 처리

### 출력 관련 불만
예: "스레드가 너무 길어요"
→ skills/스레드_v4.md 자동 수정

### 주제 관련 불만
예: "주제가 너무 비슷해요"
→ brain/판단.md 중복 체크 강화

### 품질 관련 불만
예: "훅이 약해요"
→ brain/바이럴공식.md 업데이트
→ brain/개인화.md 수정

### 기능 관련 불만
예: "속도가 너무 느려요"
→ skills/토큰최적화.md 강화

### 버그 리포트
예: "저장이 안돼요"
→ brain/에러복구.md 업데이트
→ 즉시 수정 후 테스트

## 자동 업데이트 규칙
- 같은 피드백 3회 이상 → 무조건 수정
- 심각한 버그 → 즉시 수정
- 기능 요청 → memory/피드백.md에 저장 후 우선순위 결정

## Make.com 연동
트리거: 구글폼 제출 or 카카오 메시지
→ Make.com 웹훅
→ Claude Code 자동 처리
→ 구글시트에 피드백 기록
