# 실수 방지 시스템

## 되돌릴 수 없는 작업 전 필수 확인
아래 작업 요청 시 실행 전 반드시 물어봐:
"이 작업은 되돌릴 수 없습니다. 진행할까요? (yes 입력 시 실행)"

해당 작업:
- brain/ 파일 삭제 또는 전체 교체
- Supabase 데이터 DELETE 쿼리
- .env 파일 수정
- CLAUDE.md 수정
- output/ 전체 삭제

## 백업 규칙
brain/ 파일 대규모 수정 전:
→ git commit으로 현재 상태 저장 (push 없이 로컬만)
→ 배포 없음, 언제든 git checkout으로 복원 가능

## 작업 속도 제한
한 번에 3개 이상 파일 동시 수정 금지
→ 하나씩 완료 확인 후 다음 진행

## 테스트 vs 실제 구분
테스트 작업 시 반드시 앞에 "[TEST]" 붙이기
실제 Supabase에 테스트 데이터 절대 삽입 금지
테스트용 별도 테이블 사용: _test 접미사

## 긴급 복구 명령어
준호가 "긴급복구" 입력 시:
1. memory/백업/ 최신 파일로 즉시 복원
2. 복원된 파일 목록 보고
3. 복원 전 상태 memory/백업/긴급전_[날짜]로 저장
