# GitHub + Vercel 자동 배포

## 목표
코드 변경 시 자동으로 GitHub 커밋 → Vercel 배포

## 작동 구조
Claude Code → 코드 수정
↓
GitHub API → 자동 커밋
↓
Vercel → 자동 배포
↓
웹사이트 자동 업데이트

## 실행 코드

### GitHub 자동 커밋
curl -X PUT https://api.github.com/repos/$GITHUB_REPO/contents/output/website/index.html \
  -H "Authorization: token $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "자동 업데이트: [날짜]",
    "content": "[base64 인코딩된 파일 내용]",
    "sha": "[현재 파일 SHA]"
  }'

## 디자인 자동 업데이트 주기
매주 월요일:
1. brain/디자인트렌드.md → 트렌드 수집
2. output/website/draft_[날짜].html 생성
3. junho한테 "새 버전 만들었어요" 보고
4. 승인 트리거: "배포해줘" 또는 "푸시해줘" 키워드가 있을 때만 실행
   "응", "ㅇㅇ", "좋아" 등 모호한 단어는 승인으로 인정하지 않음
5. GitHub 자동 커밋 → Vercel 자동 배포

## 승인 트리거 규칙
- 승인으로 인정되는 키워드: "배포해줘", "푸시해줘"
- 승인으로 인정되지 않는 것: "응", "ㅇㅇ", "좋아", "해줘" 등 모호한 단어
- 위 키워드 없이는 git push, vercel deploy 절대 실행 금지

## 주의사항
- .env 파일 절대 GitHub에 올리지 말 것
- .gitignore에 .env 포함 확인
- junho 승인 없이 자동 배포 금지
