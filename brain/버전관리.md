# 버전 관리 시스템

## 버전 명명 규칙
NOVA v[메이저].[마이너].[패치]
- 메이저: 시스템 구조 대변경
- 마이너: 새 기능 추가
- 패치: 버그수정/소규모 개선
현재 버전: memory/현재버전.md에 항상 기록

## 변경 전 필수 절차
brain/ skills/ CLAUDE.md 수정 전:
1. memory/버전백업/v[현재버전]_[날짜]/ 폴더에 전체 복사
2. 변경 내용 memory/변경이력.md에 기록
3. 변경 후 정상 작동 확인
4. GitHub push (커밋 메시지에 버전 명시)

## 롤백 절차
"v[버전]으로 롤백해줘" 명령 시:
1. memory/버전백업/해당버전/ 파일 확인
2. 현재 파일 임시백업 (긴급백업_[날짜])
3. 해당 버전 파일로 교체
4. 롤백 완료 보고

## 변경이력 기록 형식
memory/변경이력.md에 자동 추가:
[날짜] | v[버전] | 변경내용 | 변경이유 | 결과

## 안정 버전 지정
준호가 "이 버전 안정버전으로 지정" 명령 시:
memory/안정버전.md에 기록
→ 긴급 롤백 시 이 버전으로 우선 복구
