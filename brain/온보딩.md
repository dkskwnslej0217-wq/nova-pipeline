# 자동 온보딩 시스템

## 목표
처음 시작하는 사람이 5분 안에 시스템 실행 가능하게 안내

## 온보딩 체크리스트
실행 시 자동으로 확인:
- [ ] .env 파일 존재하는가?
- [ ] TAVILY_API_KEY 입력됐는가?
- [ ] SUPABASE_URL 입력됐는가?
- [ ] SUPABASE_KEY 입력됐는가?
- [ ] MASTER_PASSWORD 설정됐는가?
- [ ] Supabase memory 테이블 존재하는가?

## 미완료 항목 감지 시
단계별 안내 자동 출력:

### Tavily 없을 때
"1. tavily.com 접속
2. 구글 계정으로 가입
3. API Keys 클릭
4. 키 복사 후 .env에 TAVILY_API_KEY= 뒤에 입력"

### Supabase 없을 때
"1. supabase.com 접속
2. New Project 생성 (Region: Seoul)
3. Project Settings → API
4. Project URL → SUPABASE_URL에 입력
5. Publishable key → SUPABASE_KEY에 입력"

### 테이블 없을 때
"Supabase SQL Editor에서 아래 실행:
create table memory (
  id bigint generated always as identity primary key,
  created_at timestamptz default now(),
  date text, topic text, summary text, result text,
  score_trend int, score_hook int,
  score_comment int, score_total int,
  hook_type text, format_type text, length_type text
);"

## 완료 시
"✅ 모든 설정 완료! run_core.md 실행해줘"

## 신규 사용자 온보딩 (서비스 유저)

### 첫 접속 감지
Supabase users 테이블에 해당 ID 없음 → 자동 온보딩 실행

### 온보딩 순서
1단계: "NOVA에 오신 것을 환영합니다. 몇 가지 설정을 도와드릴게요. (1분 소요)"
2단계: 기본 정보 수집
- 주요 사용 목적 (콘텐츠생성/분석/자동화)
- 주력 플랫폼 (인스타/유튜브/블로그)
- 콘텐츠 스타일 (전문적/친근한/중립)
3단계: 수집 정보 → Supabase users 테이블 저장 → 이후 응답 개인화 적용
4단계: "설정 완료! '콘텐츠 만들어줘' / '분석해줘' / '도움말' 로 시작하세요"

### 재방문 사용자
Supabase 프로필 로드 → 이름으로 맞춤 인사
"다시 오셨네요! 지난번 이어서 할까요?"

### 이탈 방지
3일 이상 미접속 → memory/비활성사용자.md 기록 → 재참여 유도 메시지 준비
